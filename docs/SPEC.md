# FaceSort（分图）— 基于人物样本的照片自动归类 Mac 软件

## 1. 项目背景与目标

摄影师单次拍摄产出大量照片（数百至数千张）。本工具在拍摄结束后，根据用户提供的**人物样本照**（每人若干张），自动识别每张照片中的人物并归入以人名命名的文件夹，支持自定义命名规则（人名作为模板变量）。

## 2. 技术选型（调研结论）

| 环节 | 选型 | 理由 |
|---|---|---|
| 人脸检测 | InsightFace SCRFD（buffalo_l 模型包） | 精度/速度均衡，对侧脸、小脸鲁棒性好，优于 dlib/HOG |
| 人脸识别嵌入 | InsightFace ArcFace（512 维 embedding，buffalo_l 自带） | 开源方案中识别精度第一梯队；余弦相似度可直接做阈值匹配 |
| 推理运行时 | onnxruntime（CPU，可选 CoreML EP） | Apple Silicon 上 CPU 推理即可达 2–10 张/秒 |
| 语言/运行环境 | Python 3.12（uv 管理 venv；系统是 3.14，AI 库 wheel 可能缺失） | 生态成熟，开发速度快 |
| GUI | **Web 界面（React + Vite + Tailwind）+ pywebview 原生窗口** | 面向普通用户，精致度不受 Qt 控件限制；pywebview 用 macOS 原生 WKWebView 开窗口并提供**原生文件夹选择对话框**（纯浏览器拿不到文件夹真实路径，这是不走浏览器的关键原因）；后续可打包 .app 双击启动。CLI 保留但仅作为高级用户/自动化入口 |
| HEIC 支持 | pillow-heif | iPhone/相机 HEIC 常见 |
| 嵌入缓存 | SQLite（文件路径+mtime+size 为键） | 支持中断续跑、二次整理秒级完成 |

不选 Apple Vision framework 的原因：Vision 提供人脸检测但**不公开人脸身份嵌入 API**，无法直接做"与样本比对"；VNGenerateImageFeaturePrint 是通用图像特征，不适合人脸身份。纯 Swift 方案需自行转换 CoreML 人脸识别模型，成本高，留作 V2 备选。

## 3. 输入 / 输出约定

**样本库目录**（用户准备）：
```
samples/
  张三/        ← 目录名即人名（预设名字）
    a.jpg b.jpg   ← 每人 1–N 张清晰样本，建议正脸
  李四/
    c.jpg
```

**待整理目录**：拍摄导出的照片目录（可递归）。

**输出**：在指定输出目录（默认待整理目录下 `_sorted/`）按命名模板建文件夹并归入照片。

## 4. 验收标准

### P0（MVP 必须）
1. **CLI 可用**：`facesort run --samples <样本目录> --input <照片目录> [--output DIR] [选项]` 一条命令完成整理。
2. **单人照归类**：清晰正脸场景下，自建测试集识别正确率 ≥ 95%；不确定的照片宁可进 `_未识别` 也不错归（阈值偏保守）。
3. **三个兜底文件夹**：`_无人脸`（检测不到人脸）、`_未识别`（有人脸但与所有样本相似度低于阈值）、`_合影`（仅 group 策略时使用）。文件夹名可配置。
4. **多人照片策略**（`--multi-person`，详见 §5）：
   - `primary`（默认）：按"主体评分"选一个人归类；
   - `all`：复制到每个匹配到的人物文件夹；
   - `group`：归入 `_合影/`（可再按人名组合建子文件夹，如 `张三+李四`）。
5. **自定义命名模板**：`--folder-template` 与 `--file-template`，支持变量：
   - `{person}` 人名（多人 primary 策略下为主体人名）
   - `{persons}` 照片中所有匹配人名，`+` 连接
   - `{date}` / `{datetime}` 拍摄日期（EXIF DateTimeOriginal，缺失时用文件 mtime）
   - `{orig_name}` 原文件名（不含扩展名）、`{ext}`、`{index}`（同文件夹内序号，可格式化 `{index:03d}`）、`{similarity}` 匹配相似度
   - 默认：folder=`{person}`，file=`{orig_name}{ext}`（即只分文件夹、不改名）
6. **安全性**：默认 **copy**；`--move` 才移动；`--dry-run` 只打印归类计划（JSON/表格）不动任何文件；目标重名时自动追加 `-1/-2` 序号，绝不覆盖；原目录中的文件在 copy 模式下零改动。
7. **格式支持**：JPEG / PNG / HEIC / TIFF / WebP；非图片文件跳过并在报告中列出。
8. **嵌入缓存**：所有已算过的照片嵌入入 SQLite 缓存；重跑（如调阈值/换策略）不重复推理；中断后续跑。
9. **运行报告**：结束时输出摘要（各人张数、未识别数、无人脸数、跳过数、耗时），并写 `report.json` 到输出目录。
10. **性能**：M 系列 Mac CPU 上 ≥ 2 张/秒（不含首次模型下载）。
11. **测试**：matcher / organizer / 模板渲染 / 冲突处理有单元测试（人脸引擎用假嵌入 mock）；`pytest` 全绿。

### P1（MVP 后立即做）— Web GUI，面向普通用户，尽量不暴露 CLI 概念
12. **Web GUI（React + Tailwind，pywebview 原生窗口承载）**，验收细则：
    - 12a. **启动**：`facesort gui`（后续打包 .app 双击）打开原生 Mac 窗口，内嵌 web 界面；中文界面，零命令行知识可完成全流程。
    - 12b. **样本管理页**：可视化添加/删除人物；为每个人拖拽或选择样本照片；界面显示检测到的人脸裁剪缩略图；样本照检测不到人脸时即时红字提示（避免用户带着坏样本开跑）。
    - 12c. **整理配置页**：原生对话框选择照片目录/输出目录（pywebview 桥调 NSOpenPanel，拿真实路径）；多人策略用大白话单选卡片（"归入主要人物 / 每人都存一份 / 单独放进合影"）+ 示意图；阈值用"宽松—严格"滑杆（隐藏余弦相似度术语，高级面板可看数值）；命名规则提供预设模板下拉（如 `人名/原文件名`、`人名/日期_序号`）+ 自定义输入框 + **实时文件名预览**。
    - 12d. **预览（dry-run 可视化）**：正式执行前展示归类计划——按人物分组的缩略图墙、各组张数、未识别/无人脸列表；用户确认后才动文件。
    - 12e. **执行页**：总进度条 + 当前照片缩略图 + 速度/剩余时间；可取消（已完成部分保留）。
    - 12f. **结果页**：摘要卡片（各人张数等）；**歧义照片人工复核**——对报告中的歧义列表逐张展示候选人物，一键指派并即时移动；"在 Finder 中打开"按钮。
    - 12g. 前端构建产物（静态文件）随包分发，运行时不依赖 Node；界面自适应窗口大小，支持暗色模式。
13. 每人多样本时的原型策略：默认对每人所有样本嵌入取"匹配时逐张比对取最大相似度"（优于简单平均，抗坏样本）。
14. `--min-face` 最小人脸尺寸（像素）过滤路人/背景小脸，默认 40px。
15. RAW（CR3/NEF/ARW/DNG）：读取内嵌 JPEG 预览做识别，归类移动原 RAW 文件；同名 RAW+JPEG 成对时视作一组一起归类。

### P2（后续）
16. 打包为 .app（py2app/PyInstaller）+ 图标。
17. 无样本模式：自动聚类未知人物成"人物1/人物2"，用户回填名字。
18. 视频、Live Photo 支持。

## 5. 多人照片主体判定（primary 策略）

一张照片检测到多张人脸时，对每张与样本库匹配成功（相似度 ≥ 阈值）的人脸计算主体评分：

```
score = w_area * 面积分 + w_center * 中心分 + w_sim * 相似度分
```
- **面积分**：该人脸框面积 / 照片中最大人脸框面积（拍摄主体通常最近/最大）
- **中心分**：1 − 人脸中心到画面中心的归一化距离（构图主体常在中间；按三分法适当放宽，用高斯衰减而非线性）
- **相似度分**：与样本的余弦相似度（归一化到 0–1）
- 默认权重 `w_area=0.45, w_center=0.25, w_sim=0.30`，CLI 可通过 `--weights` 覆盖

取分最高者为主体。**只在匹配成功的人脸中选主体**——画面中央的路人不会因为居中而胜过旁边匹配到样本的人。若无任何人脸匹配成功 → `_未识别`。

## 6. 架构

```
facesort/
├── pyproject.toml            # uv 管理，Python 3.12
├── facesort/
│   ├── core/
│   │   ├── engine.py         # FaceEngine：insightface 封装（检测+嵌入），惰性加载模型
│   │   ├── matcher.py        # 样本库构建、余弦匹配、主体评分（纯逻辑，可 mock 测试）
│   │   ├── organizer.py      # 归类计划生成 Plan → 执行（copy/move/dry-run/冲突改名）
│   │   ├── templates.py      # 命名模板解析与渲染（{person} 等变量）
│   │   ├── cache.py          # SQLite 嵌入缓存
│   │   └── models.py         # dataclass：Face, Photo, MatchResult, PlanItem, Config
│   ├── cli.py                # CLI 入口（typer；面向高级用户/自动化，普通用户走 GUI）
│   ├── gui/                  # P1：pywebview 窗口 + Python↔JS 桥（js_api：选目录、跑 pipeline、
│   │                         #   推送进度 evaluate_js）；web 前端为 React+Vite+Tailwind，
│   │                         #   构建产物静态打包，运行时不依赖 Node
│   └── report.py             # 摘要与 report.json
├── webui/                    # 前端源码（React+Vite+Tailwind），build 输出到 facesort/gui/static/
└── tests/                    # 单测：matcher/organizer/templates/cache（mock 引擎）
```

关键设计：
- **计划/执行分离**：先生成完整 `Plan`（每张照片 → 目标路径 + 原因），dry-run 即打印 Plan；GUI 和 CLI 共用。
- **引擎与逻辑解耦**：`matcher`/`organizer` 不依赖 insightface，单测用假嵌入向量。
- **阈值**：ArcFace 余弦相似度默认阈值 0.40（保守），`--threshold` 可调，报告中打印每张的相似度便于用户调参。

## 7. 边界条件（明确行为）

| # | 边界情况 | 行为 |
|---|---|---|
| 1 | 照片无人脸 | 归入 `_无人脸/` |
| 2 | 有人脸但都低于阈值 | 归入 `_未识别/` |
| 3 | 人脸过小（< min-face） | 该人脸忽略；若全部忽略视同无人脸 |
| 4 | 同一人脸与两个人的样本都超阈值 | 取相似度最高者；若前两名相似度差 < 0.05 记入报告"歧义列表"供人工复核 |
| 5 | 样本照里本身有多张脸 | 取最大人脸作为该人样本，并在日志中警告 |
| 6 | 样本目录为空 / 某人目录无有效图 | 启动即报错退出，不做半程整理 |
| 7 | 损坏/截断的图片文件 | 跳过，记入报告 skipped，不中断整个批次 |
| 8 | 目标文件已存在 | 追加 `-1/-2` 序号，绝不覆盖 |
| 9 | 输出目录在输入目录内部（默认 `_sorted/`） | 扫描时排除输出目录，避免自吞 |
| 10 | 重复运行（幂等） | copy 模式下重复运行借助缓存与已存在检测不产生重复副本（相同来源+目标跳过） |
| 11 | EXIF 无拍摄时间但模板用了 `{date}` | 回退文件 mtime，并在报告标注 |
| 12 | 人名含 `/` 等非法路径字符 | 渲染文件夹名时替换为 `_` |
| 13 | 中断（Ctrl-C） | 已复制的文件保留，缓存已写入；重跑续做 |
| 14 | 全部照片都是合影且策略为 primary | 正常按主体归类；报告中给出多人照片计数提示可改用 all/group |
| 15 | 移动模式跨磁盘卷 | 用 copy+verify+delete 语义（shutil.move 已处理），失败保原文件 |

## 8. 里程碑

- **M1**：core + CLI，P0 全部验收项，pytest 全绿，真实照片冒烟通过。core 的 pipeline 必须支持进度回调（per-photo callback）与取消令牌，供 GUI 复用。
- **M2（已完成）**：Web GUI（§4 P1 第 12 项全部细则）。
- **M3（已完成）**：(1) RAW 支持（CR2/CR3/NEF/ARW/DNG 读内嵌预览识别 + 同名 RAW+JPEG 配对同归，rawpy）；(2) 无样本聚类模式（`facesort cluster` + GUI 模式切换，自动分「人物1..N」，聚类中心作合成样本库复用归类逻辑）；(3) 打包 .app（PyInstaller，`packaging/FaceSort.spec` + `build_app.sh`，产出 dist/FaceSort.app 双击即用，模型首次运行时下载；冻结环境自检验证检测+识别正常）。
